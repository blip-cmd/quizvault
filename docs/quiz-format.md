# Quiz Format & Import Parsing

QuizVault operates on standard portable JSON arrays to populate the app structurally.

To properly load custom study materials generated by Large Language Models or pulled into the app manually, the data strictly must match the target schema below.

## Schema Example

```json
{
  "title": "My Custom Exam - Chapter 1",
  "questions": [
    {
      "question": "Which of the following describes standard Offline-First design?",
      "options": [
        "A constant WebSocket connection to the host",
        "Utilizing Service Workers to cache assets and bypass the network entirely",
        "Running Docker containers locally on user hardware"
      ],
      "correctIndex": 1,
      "explanation": "Offline-first relies firmly on the App Cache and Service Workers logic holding the UI layer available without any actual TCP connection required."
    }
  ]
}
```

## Field Constraints
*   **title** *(Optional String)*: The display name mapping. Highly recommended. If omitted, defaults to "Untitled Quiz".
*   **questions** *(Required Array)*: An array containing the node questions.
    *   **question** *(Required String)*: The raw question text itself.
    *   **options** *(Required Array of Strings)*: An array containing the available options to choose from. Must encompass at least two options.
    *   **correctIndex** *(Required Integer)*: The 0-based array index identifying the correct string inside the `options` array.
    *   **explanation** *(Optional String)*: Displayed either when the user guesses incorrectly initially, or manually via a button toggle if answered correctly. Supports mild textual rendering.
